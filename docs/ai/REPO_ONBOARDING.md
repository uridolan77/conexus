# Repo Onboarding Checklist

1. Confirm the repo is Conexus or another repo intentionally adopting the Conexus agent-os package.
2. Identify runtime surfaces: backend, frontend, database, deployment target, and provider integrations.
3. Verify `.agent-os/profile.yml` matches the repo's real paths and validation commands.
4. Locate or create the Conexus source-of-truth docs under `docs/product`, `docs/architecture`, and `docs/specs`.
5. Confirm hooks and protected-path policy work locally.
6. Ask the first agent to summarize rules and the smallest safe Conexus slice before editing files.
